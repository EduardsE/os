RAID 0

RAID 0 consists of striping data and it doesn't use redundancy.
Therefore, failure of one drive causes the loss of the entire RAID 0 volume.

RAID 1

RAID 1 consists of data mirroring, identical data is written to two drives.
If there's a failure in one drive, data can be read from the other one.

RAID 5

RAID 5 consists of striping with distributed parity.
Parity information is spread across the drives along with the data and can be used to regenerate any missing data in case of one disk failure.

RAID 6

RAID 6 consists of striping with double distributed parity.
Unlike in RAID 5, data can be recovered even if two drives fail.

RAID 10

RAID 10 combines RAID 0 and RAID 1 - striping data between two drives and mirroring that.
If one drive fails, then data still can be successfully read from the other drives. If 2 drives fail, you might get lucky and data still could be read.

How it works?

You can select the RAID level in top bar, by default RAID 0 is shown.
Start by writing data in drives (Write data button).
Then there are 3 options:

  • Hard drive failure (it is randomized);
  • Reset - if you want to start again;
  • Read data - we will tell you if it's possible.

Enjoy!
